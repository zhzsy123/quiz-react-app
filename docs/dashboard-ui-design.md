# Dashboard UI Design

本文记录当前首页的视觉目标、板块分布和交互约束，作为后续首页迭代的统一参考。它只描述首页，不覆盖工作区答题页、AI 出题对话框或存储实现。

## 设计目标

- 首页应呈现为稳定、成熟的学习平台控制台，而不是功能清单。
- 首屏必须保留三个最高频动作：进入科目、下载资料、更新 API Key。
- 档案切换、新建档案、错题/收藏/历史入口应在首屏内完成，但不能压过主视觉。
- JSON 工作流说明要清楚，但只能作为辅助带，不应抢占科目入口。

## 页面结构

### 1. 首屏展示区

- 左侧：
  - 产品标题
  - 一句价值说明
  - 三个核心按钮
  - 四个摘要统计
- 右侧：
  - 当前档案面板
  - 快捷入口面板

### 2. 科目模块区

- 用三列卡片展示当前支持科目。
- 每张卡片只展示：
  - 科目名称
  - 一句描述
  - 历史参与次数
  - 平均正确率
- 点击整张卡片进入科目题库页。

### 3. JSON 工作流区

- 以一条横向 band 展示：
  - 试卷原文
  - AI 输出 JSON
  - 导入练习
- 仅保留一个“下载资料”按钮，不重复堆叠说明文字。

### 4. 底部摘要区

- 最近一次考试
- 总刷题量
- 当前重点（错题总量）

## 视觉风格

- 背景：深色渐变，辅以局部光晕。
- 卡片：统一圆角、细边框、轻玻璃感。
- 强调色：
  - 蓝色用于主按钮和关键入口
  - 绿色用于状态确认
  - 红色只用于危险状态
- 标题：大字号、紧凑字距。
- 说明文字：对比度略低于标题，避免满屏发白。

## 交互反馈

- 主按钮 hover 只做轻微抬升或描边增强。
- 快捷入口 hover 可轻微横向位移，提示可跳转。
- 下载资料使用统一弹层承载，按科目分组显示。
- 更新 API Key 只保留一个按钮入口，避免首屏额外出现泛白信息区。
- 所有弹窗关闭后应保持首页原状态，不重置首屏内容。

## 文案原则

- 标题尽量短。
- 说明尽量一句话说清。
- 下载项标题必须与科目名对应，例如“英语试题 JSON 解析规范文档”。
- 下载项副文案必须说明支持题型，不写空泛口号。
- 首页不重复解释题型协议细节，这些内容放在独立文档中。

## 响应式原则

- 桌面端：首屏双栏，科目区三列。
- 中等宽度：首屏改为单栏，科目区自动两列。
- 移动端：首屏按钮纵向排列，科目卡片和摘要卡全部单列显示。
